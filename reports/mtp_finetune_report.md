# MTP 微调结果汇报 — 域内涨、域外基本守住（ALLaVA + MMStar）

**日期**：2026-06-13　|　**模型**：Qwen3.5-9B（多模态 verifier）　|　**方法**：投机解码 MTP 头微调

## 结论（TL;DR）

在 100k Qwen 蒸馏 ALLaVA 上微调 Qwen3.5-9B **自带的原生 MTP 头**：

> **域内（ALLaVA）：平均接受长度 +7.7%、吞吐 +6.1%，全面超过原生 MTP。**
> **域外（MMStar，未训练）：首位接受率基本持平（−0.5%），平均接受长度小幅回退 −4.6%（非崩溃）。**

即：微调把 MTP 头**特化到了目标域**——域内收益明确，域外有可控的小幅代价。面向 ALLaVA 类多模态域部署是净赚；若要做通用 serve，建议训练时掺入少量通用数据。

## 结果

**① 域内 ALLaVA（目标场景，越高越好）**

| 指标 | 原生 MTP | 微调 MTP | 变化 |
|---|---:|---:|---:|
| **平均接受 token / draft** | 2.916 | **3.142** | **+7.7%** |
| 输出吞吐 tok/s | 66.4 | **70.4** | **+6.1%** |
| 首位接受率 | 0.838 | 0.844 | +0.7% |

**② 域外 MMStar（遗忘检查，希望"基本持平"）**

| 指标 | 原生 MTP | 微调 MTP | 变化 |
|---|---:|---:|---:|
| 平均接受 token / draft | 2.822 | 2.694 | −4.6% |
| 输出吞吐 tok/s | 69.8 | 65.9 | −5.6% |
| **首位接受率** | 0.828 | 0.824 | **−0.5%（持平）** |

- 两数据集、两路均 **128/128** 成功完成。
- **首位接受率域外几乎不掉**（0.828→0.824）→ draft 的"下一个 token"质量没退化；域外的小幅回退只在更深位置，是模型更偏 ALLaVA 风格的结果，非能力丢失。
- 对照：之前 kl_div 方案的域外遗忘约 −13%（0.87×），本次 −4.6% 明显更轻。

## 实验设置（公平、无数据泄漏）

| 项 | 设置 |
|---|---|
| 训练 | 100k Qwen 自蒸馏 ALLaVA；bf16 + LR 3e-5；checkpoint 按验证 loss 选 |
| 域内验证集 | 该 100k 的**后 10%（held-out 9972 条）**，训练未见过 → 测泛化非记忆 |
| 域外验证集 | MMStar（模型从未训练）；原生 MTP 复现已知基线 **0.828 / 2.822**，确认口径正确 |
| 对比口径 | 原生 = serve 原始 verifier（自带头）；微调 = serve 缝合后检查点（微调头）。**两路同方法 `qwen3_5_mtp`、同投机步数 7、同验证集，唯一差别是 MTP 头权重** |

## 关键工程产出

- **打通"微调 MTP 头如何上线"**：vLLM 无法直接挂裸 MTP 检查点（头嵌在 verifier 权重内），用
  `scripts/stitch_mtp.py` 把微调层**缝合**回完整 verifier 再 serve；自检确认微调头已生效。
- 一键评测：`test_mtp_allava_orig_vs_trained.sh`（域内）+ `test_mtp_mmstar_orig_vs_trained.sh`（域外）。
